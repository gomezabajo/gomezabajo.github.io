We propose a model-driven engineering approach that automates the construction of metamorphic testing (MT) environments. Given a metamodel that captures the domain concepts and a description of the domain execution environment, the approach generates an MT environment that supports the complete testing process. This support includes defining domain-specific metamorphic relations (MRs), evaluating them, producing detailed test reports, and automatically generating follow-up test cases through search-based techniques.
